# Знакомство с IaaS, PaaS, SaaS сервисами в облаке на примере Amazon Web Services (AWS). Создание сервисной модели.

## Описание работы

### Цель работы:
Знакомство с облачными сервисами. Понимание уровней абстракции над инфраструктурой в облаке. Формирование понимания типов потребления сервисов в сервисной-модели.

### Дано(Вариант 1):
1. Слепок данных биллинга от провайдера после небольшой обработки в виде SQL-параметров. Символ % в начале/конце означает, что перед/после него может стоять любой набор символов.
2. Образец итогового соответствия, что желательно получить в конце.
### Необходимо:
1. Импортировать файл .csv в Excel или любую другую программу работы с таблицами. Для Excel делается на вкладке Данные – Из текстового / csv файла – выбрать файл, разделитель – точка с запятой.
2. Распределить потребление сервисов по иерархии, чтобы можно было провести анализ от большего к меньшему (напр. От всех вычислительных ресурсов Compute дойти до конкретного типа использования - Выделенной стойка в датацентре Dedicated host usage).
3. Сохранить файл и залить в соответствующую папку на Google Drive.

### Алгоритм работы:
Сопоставить входящие данные от провайдера с его же документацией. Написать в соответствие колонкам справа значения 5 колонок слева, которые бы однозначно классифицировали тип сервиса. Для столбцов IT Tower и Service Family значения можно выбрать из образца.

## Ход работы

Изначально был предоставлен слепок данных биллинга AWS, содержащий сведения о потреблении различных облачных сервисов.

Данные были импортированы в таблицу (формат CSV) и проанализированы в соответствии с документацией Amazon Web Services. Для уточнения классификации также использовались дополнительные источники официальной информации. На основе анализа каждому сервису было присвоено место в иерархии сервисной модели — от уровня инфраструктуры до конкретного типа использования.

### Краткое описание классифицированных сервисов

1. **Amazon Translate**  
   Сервис машинного перевода текстов. Используется для локализации приложений, создания субтитров и автоматического перевода контента.

2. **Amazon Transcribe**  
   Сервис автоматического преобразования аудио в текст с использованием технологий распознавания речи. Применяется для расшифровки аудио- и видеоматериалов.

3. **Amazon S3 (Simple Storage Service)**  
   Сервис объектного хранения данных, обеспечивающий масштабируемость, высокую доступность и надежность хранения.

   Дополнительно были рассмотрены классы хранения:
    - **Amazon Glacier / Deep Archive** — решения для долговременного и экономичного хранения редко используемых данных.
    - **S3 Standard-IA** — хранение данных с низкой частотой доступа, но с быстрым восстановлением.
    - **One Zone-IA** — хранение данных в одной зоне доступности, что делает его более экономичным.
    - **Intelligent-Tiering** — автоматическое распределение данных между уровнями хранения в зависимости от частоты обращения.

   В таблице для этих сервисов столбец «Storage Classes» оставлен обобщённым, поскольку конкретные классы хранения могут различаться в зависимости от параметров использования.

4. **Amazon Redshift**  
   Облачная аналитическая база данных, предназначенная для хранения и обработки больших объемов данных (до петабайтов) и выполнения SQL-запросов для анализа.

5. **Amazon Simple Notification Service (SNS)**  
   Сервис обмена сообщениями и уведомлениями, обеспечивающий взаимодействие между микросервисами и доставку уведомлений пользователям через SMS, электронную почту и другие каналы.

6. **AWS CodePipeline**  
   Сервис для организации процессов непрерывной интеграции и доставки (CI/CD), который автоматизирует этапы сборки, тестирования и развертывания приложений.

7. **AWS CodeBuild**  
   Сервис сборки программного обеспечения, предназначенный для компиляции исходного кода, запуска тестов и создания готовых артефактов для последующего развертывания.

8. **Amazon Machine Learning**  
   Сервис, позволяющий строить, обучать и развертывать модели машинного обучения, предназначенные для анализа данных и поиска закономерностей.

9. **Amazon Polly**  
   Сервис синтеза речи, преобразующий текст в реалистичное аудио с использованием технологий машинного обучения.

10. **AWS Directory Service**  
    Сервис управления каталогами пользователей и ресурсов, который обеспечивает интеграцию локальных каталогов с облачной инфраструктурой и централизованное управление доступом.

После сопоставления всех сервисов с документацией и их классификации была сформирована итоговая таблица с иерархией сервисов AWS.  
Каждому элементу биллинга было присвоено соответствующее значение по уровням:
- **IT Tower** (направление),
- **Service Family** (семейство сервисов),
- **Service** (тип сервиса),
- **Service Type** (конкретный вид использования).

Полученная таблица отражает структуру потребления облачных сервисов и позволяет проводить дальнейший анализ распределения затрат и уровней абстракции.

В ходе выполнения лабораторной работы были изучены принципы классификации облачных сервисов Amazon Web Services по уровням абстракции: IaaS, PaaS и SaaS.  
На основе данных биллинга выполнено сопоставление сервисов с их функциональными категориями и определены соответствующие уровни потребления в сервисной модели.

В результате работы сформировано понимание различий между уровнями инфраструктуры, платформы и программного обеспечения как сервиса.  
Были получены практические навыки классификации облачных ресурсов, а также построения иерархии сервисов для анализа потребления в облачной среде.
