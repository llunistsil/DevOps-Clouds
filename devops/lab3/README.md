# Лабораторная работа №3: CI/CD — Как не стать Васей

## Команды для локального тестирования

```bash
brew install act
act -W .github/workflows/bad-ci.yml -n
act -W .github/workflows/good-ci.yml -n
```

---

## Метрики и результаты

| Метрика          | Плохой CI/CD         | Хороший CI/CD |
| ---------------- | -------------------- | ------------- |
| Время выполнения | ~35 мин              | ~12 мин       |
| Надежность       | 40% успешных деплоев | 95% успешных  |
| Безопасность     | Секреты в коде       | Secrets API   |

---

## Уроки из истории про Васю

* Делай бэкапы (и бэкапы бэкапов)
* Не деплой в пятницу
* Staging — твой лучший друг
* Автоматизация ≠ бездумность
* Документируй всё, включая откаты

---

## Выводы

Хороший CI/CD — это система защиты от человеческих ошибок.
Плохой CI/CD — это ticking time bomb.

**Принципы:**

* Версионируй всё
* Не храни секреты в коде
* Разделяй этапы
* Тестируй по-настоящему
* Используй кеширование
* Требуй подтверждения для прода
* Мониторь после деплоя
* И не деплой в пятницу!
