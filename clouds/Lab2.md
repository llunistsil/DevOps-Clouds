# Сравнение сервисов Amazon Web Services и Microsoft Azure. Создание единой кросс-провайдерной сервисной модели.

Вариант 1

## Описание работы

### Цель работы

Получение навыков аналитики и понимания спектра публичных облачных сервисов без привязки к вендору.  
Формирование комплексного видения облачной инфраструктуры и принципов классификации сервисов различных провайдеров.

### Дано(Вариант 1):

1. Данные лабораторной работы 1.
2. Слепок данных биллинга от провайдера после небольшой обработки в виде SQL-параметров. Символ `%` в начале/конце означает, что перед/после него может стоять любой набор символов.
3. Образец итогового соответствия, который необходимо получить в конце работы.

### Необходимо:

1. Импортировать файл `.csv` в Excel или любую другую программу для работы с таблицами.  
   Для Excel используется вкладка **Данные → Из текста/CSV**, разделитель — точка с запятой.
2. Распределить потребление сервисов по иерархии, чтобы можно было провести анализ от общего к частному (например, от группы вычислительных ресурсов *Compute* до конкретного типа использования — *Dedicated host usage*).  
   При этом необходимо сохранять логическую концепцию, выработанную в **Лабораторной работе 1**.
3. Сохранить готовый файл и разместить его в соответствующей папке на Google Drive.

### Алгоритм работы:

1. Сопоставить входящие данные от провайдера с официальной документацией Microsoft Azure.
2. В таблице определить пять классификационных колонок (например: *IT Tower*, *Service Family*, *Service*, *Service Type*, *Notes*).
3. Для столбцов *IT Tower* и *Service Family* значения можно выбрать из образца предыдущей лабораторной работы.
4. При классификации сохранять единые принципы, использованные в работе с AWS.  
   Например, если сервис машинного обучения ранее относился к группе «Вычислительные мощности» и «Облачные сервисы», то при анализе данных Azure следует придерживаться той же логики.

## Ход работы

Исходные данные были импортированы в таблицу и проанализированы с использованием документации Microsoft Azure.  
Целью анализа было сопоставление сервисов Azure с их аналогами из AWS и включение их в единую кросс-провайдерную сервисную модель.

### Описание классифицированных сервисов

- **Azure Analysis Services**  
  Сервис, который позволяет создавать и управлять моделями данных в облаке с использованием языка DAX (Data Analysis Expressions).  
  Применяется для анализа, агрегации и визуализации данных из различных источников при обеспечении контроля доступа и безопасности.

- **Azure Data Factory**  
  Платформа для интеграции данных, которая позволяет создавать, управлять и отслеживать процессы ETL (извлечение, преобразование, загрузка).  
  Используется для объединения данных из различных источников и подготовки их для аналитики.

- **Azure Database Services**  
  Набор сервисов для развертывания и управления реляционными и нереляционными базами данных (включая MySQL, PostgreSQL, SQL Database и др.).  
  Обеспечивает автоматическое масштабирование, резервное копирование и обновления без участия пользователя.

- **Azure Redis Cache**  
  Сервис кэширования данных в оперативной памяти, повышающий производительность приложений и уменьшающий задержки при обращении к данным.  
  Позволяет снизить нагрузку на основную базу данных.

- **Azure CDN (Content Delivery Network)**  
  Сервис для ускоренной доставки контента пользователям по всему миру за счет кэширования данных на серверах, расположенных в различных регионах.  
  Уменьшает время отклика и повышает устойчивость систем к нагрузкам.

- **Azure Batch**  
  Сервис для запуска и управления большими параллельными вычислительными задачами в облаке.  
  Применяется для обработки данных, рендеринга видео и выполнения научных вычислений.

- **Azure Virtual Machines**  
  Сервис, предоставляющий виртуальные машины с возможностью выбора операционных систем (Windows, Linux) и параметров масштабирования.  
  Позволяет гибко управлять вычислительными ресурсами и резервированием.

- **Azure Data Box**  
  Аппаратно-программное решение для переноса больших объемов данных в облако Azure.  
  Позволяет безопасно копировать данные с локальных систем на физическое устройство, которое затем доставляется в центр обработки данных Microsoft.

- **Azure Key Vault**  
  Сервис безопасного хранения и управления секретами, ключами шифрования и сертификатами.  
  Обеспечивает централизованный контроль доступа и защиту конфиденциальных данных.

- **Azure Scheduler**  
  Сервис планирования и автоматизации выполнения задач в облаке.  
  Используется для запуска процессов по расписанию и автоматического управления сервисами.

- **Azure Sentinel**  
  Платформа обеспечения безопасности и анализа угроз (SIEM/SOAR), объединяющая данные из различных источников.  
  Позволяет выявлять, анализировать и реагировать на инциденты безопасности как в облаке, так и в локальной инфраструктуре.

После анализа все сервисы были классифицированы по соответствующим категориям и добавлены в итоговую таблицу, включающую данные как AWS, так и Microsoft Azure.  
Это позволило сформировать единую кросс-провайдерную модель, отражающую универсальные принципы организации облачных сервисов независимо от поставщика.
